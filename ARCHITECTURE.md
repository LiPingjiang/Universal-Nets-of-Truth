# 方隅元筌 · 项目架构与书写规范

## 一、全局架构原则

### 1.1 三大文明传统分层

本书采用全球高校通用、欧美哲学界完全认可的标准划分：

| 层级 | 名称 | 说明 |
|------|------|------|
| 顶层 | 西方哲学（Western Philosophy） | 古希腊源头，经中世纪、近代、当代 |
| 顶层 | 中国哲学（Chinese / East Asian Philosophy） | 儒道源头，经魏晋玄学、宋明理学 |
| 顶层 | 印度哲学（Indian / South Asian Philosophy） | 吠陀传统，经婆罗门教、佛教、六派哲学 |
| 次级 | 非西方哲学（Non-Western Philosophy） | 伊斯兰、非洲、拉美等，归入交汇脉络 |

**注意**：分析哲学（Analytic）与欧陆哲学（Continental）的划分仅限20世纪后西方哲学内部使用，不用于全球文明分类。

### 1.2 目录组织规则

```
{序号}_{流派或时期名称}/
├── README.md              # 该流派/时期总览
├── {人物或著作}_原文.md    # 原文/核心文本（如有）
├── {人物或著作}_解读_学术.md
├── {人物或著作}_解读_白话.md
└── 附录/
    └── 相关图表、年表等
```

### 1.3 文件四件套（单篇体例）

每篇核心文本或每位核心哲学家配备四件套：

1. **`XX_原文.md`**：原始语言文本（如有）+ 双轨汉译对照
2. **`XX_解读_学术.md`**：学术解读，术语精确，含文献考证、思想史定位
3. **`XX_解读_白话.md`**：白话解读，正文不出现音译术语，术语沉脚注
4. **Excel 行**：元信息骨架（待后续统一建表）

## 二、双轨汉译规范

### 2.1 甲译（白话意译）

- **加粗规则**：主翻译正文一律用 Markdown 加粗（`**...**`），包括缘起叙述句和所有偈颂/引文
- **括注规则**：首次出现用"熟语（原文为X，意思是……）"夹注；同段后续重复用熟语简称
- **脚注标记**：放在加粗之外，写成 `**正文**[^x]**正文**`
- **熟语优先**：原词（尤其音译词）降为括注，正文以现代熟语/大白话为主角

### 2.2 乙译（学术直译）

- 不加粗
- 术语精确，逐句对应原文
- 以忠实为第一原则，不要求白话

### 2.3 脚注规范

- 使用具名标签，不用数字序号（避免甲乙两译引用同一概念时编号混乱）
- 命名取原文关键词简写，如 `[^nibbāna]`、`[^saccā]`、`[^brahma]`
- 内容：术语原文 + 字面本义 + 简要说明 + 附录链接

## 三、GLOSSARY 五分区体系

`附录/GLOSSARY.md` 统一在一个文件中，保持链接路径稳定：

1. **一、术语** — 哲学专业术语
2. **二、人物** — 历史人物，含真名/绰号/各语言对应/生平/与核心思想关系
3. **三、地名** — 地名含古今地理对应
4. **四、文词** — 典籍专用文词，用熟语转换说明
5. **五、文化背景** — 古文化/宗教/社会背景

### 3.1 熟语速读体例

- 凡字面与本义有隔阂、易望文生义的条目，开头加一行 **熟语速读**，用现代人现代行文风格的大白话一句点破
- 不用另一堆古词解释古词
- 多字拼合名称逐字拆解，说明每字指人/物/地
- 主动用 ⚠️ 点破常见误解

### 3.2 源语言夹注体例

音译/绰号类条目的**标题括号内**直接夹注源语言与本义：

```
### 词条（原词，语言，意为"……"）
```

示例：佛陀（Buddha，巴利语/梵语，意为"觉醒者"）

## 四、古今熟语对照表

`附录/古今熟语对照.md` 记录词汇"当时是大白话 → 汉译后变难懂 → 今天怎么说"的转化过程。

**五列结构**：词汇 | 原词（语言）| 当时听到的感觉 | 为何汉译后变难懂 | 今天的大白话

**分组**：核心概念 / 修行心性 / 人物称谓 / 文体场景 / 宗教生态

---

## 五、括注用原文词原则（硬规则）

1. 甲译正文用熟语/本名；括注（解释部分）**必须填真正的原文词**，绝不能再做一次意译替换
2. 写"原文为X"之前，一定要回看该篇的原文，确认对应的真正原词
3. 读者对本名反而清楚，对音译反而易误解；正文用本名/熟语是对的，括注更要用真原词
