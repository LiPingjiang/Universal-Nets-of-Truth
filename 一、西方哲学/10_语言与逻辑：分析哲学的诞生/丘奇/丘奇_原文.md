# 丘奇 · 原文

## 一、生平与著作

阿隆佐·丘奇（Alonzo Church, 1903-1995），美国数学家与逻辑学家，生于华盛顿特区，逝于俄亥俄州哈德逊。1921年入普林斯顿大学，1927年获博士学位。1929年至1967年任教于普林斯顿大学数学系，1967年后转任加州大学洛杉矶分校（UCLA）哲学与数学系，直至1990年退休。

丘奇是20世纪数理逻辑的核心人物之一。他指导的博士研究生包括艾伦·图灵（Alan Turing）、斯蒂芬·克林（Stephen Kleene）、巴克利·罗瑟（J. Barkley Rosser）、利昂·亨金（Leon Henkin）、迈克尔·拉宾（Michael Rabin）与达纳·斯科特（Dana Scott）等人，几乎构成此后数十年逻辑学与理论计算机科学的半壁江山。1936年，丘奇创办《符号逻辑杂志》（*Journal of Symbolic Logic*），并长期担任主编，使之成为该领域的旗舰刊物。

核心著作与论文：

- "A Set of Postulates for the Foundation of Logic" (1932, 1933) —— 最初提出λ演算的雏形，试图为逻辑建立一套基于函数的公理系统。
- "An Unsolvable Problem of Elementary Number Theory" (1936) —— 系统引入λ演算，并证明存在不可判定的初等数论问题。
- "A Note on the Entscheidungsproblem" (1936) —— 证明一阶逻辑的判定问题（Entscheidungsproblem）不可解，即"丘奇定理"。
- *Introduction to Mathematical Logic* (1956) —— 数理逻辑的标准教材，影响深远。

## 二、核心文本

### 2.1 λ演算（Lambda Calculus）

#### 2.1.1 λ抽象与函数应用

丘奇在1936年的论文中系统阐述了λ演算的形式体系。其核心思想是将一切计算归结为函数的抽象（abstraction）与应用（application）。

> **"We select a particular notation for functions, namely the notation λx·M, where M is a function of x (or a formula containing x), and λx·M denotes the function whose value for each argument x is M."**
>
> **"The function λx·x is the identity function; λx·x² is the squaring function; λf·λx·f(f(x)) is the function which, when applied to a function f, yields the function which applies f twice."**

论证过程：丘奇首先定义了"变量""函数""应用"三个基本概念。一个表达式形如 `λx.M` 表示一个以 x 为参数、函数体为 M 的匿名函数。当这个函数被应用于一个参数 N 时，即 `(λx.M) N`，将 M 中所有自由出现的 x 替换为 N，这一过程称为β归约（β-reduction）。例如：`(λx.x²) 5 → 5² → 25`。

**甲译（白话意译）：** **我们选一套专门记函数的符号，就是 λx·M。这里 M 是跟 x 有关的表达式，而 λx·M 就代表那个"你给我一个 x，我就还你一个 M"的函数。λx·x 就是恒等函数——你给我啥我还你啥；λx·x² 就是平方函数——你给我 3 我还你 9。当你把一个函数拿去用在一个值上，就把函数体里的变量换成那个值，算出来就行。**

乙译（学术直译）：吾人选定一特定函数记法，即 λx·M，其中 M 为 x 之函数（或含 x 之公式），λx·M 乃表一函数，其对于各变元 x 之值即为 M。函数 λx·x 为恒同函数；λx·x² 为平方函数；λf·λx·f(f(x)) 则为如下之函数：当其应用于函数 f 时，即产出将 f 施行两次之函数。

#### 2.1.2 丘奇数（Church Numerals）

> **"The positive integer n is represented by the function λf·λx·fⁿ(x), where fⁿ(x) denotes the n-fold iteration of f applied to x."**
>
> **"Thus 1 is λf·λx·f(x), 2 is λf·λx·f(f(x)), 3 is λf·λx·f(f(f(x))), and so on."**

论证过程：丘奇的天才之处在于，他证明了自然数可以完全用函数来编码。数字 n 被表示为"将一个函数 f 迭代 n 次"的高阶函数。例如，数字 2 就是 `λf.λx.f(f(x))`——接受任意函数 f 和参数 x，返回 f 对 x 施行两次的结果。在此基础上，加法、乘法、指数运算等算术运算均可纯以λ演算表达：

- 后继函数（加一）：`SUCC = λn.λf.λx.f(n f x)`
- 加法：`PLUS = λm.λn.λf.λx.m f (n f x)`
- 乘法：`MULT = λm.λn.λf.m(n f)`

**甲译（白话意译）：** **自然数 n 就表示成"把一个函数套 n 次"。1 就是套一次，2 就是套两次，3 就是套三次。你看，根本不需要"数"这个概念——光靠函数和函数的嵌套，就能把所有的自然数都造出来。加法就是"先套 m 次再套 n 次"，乘法就是"把套 n 次这件事本身套 m 次"。一切算术，归根到底都是函数的复合。**

乙译（学术直译）：正整数 n 以函数 λf·λx·fⁿ(x) 表示之，其中 fⁿ(x) 表 f 对 x 施行 n 次迭代。故 1 即 λf·λx·f(x)，2 即 λf·λx·f(f(x))，3 即 λf·λx·f(f(f(x)))，余可类推。

#### 2.1.3 λ演算的普遍性

> **"The class of λ-definable functions of positive integers is the same as the class of recursive functions, and this class includes all functions whose values can be effectively calculated."**
>
> **"The lambda calculus is a universal model of computation: every effectively calculable function can be expressed as a lambda expression."**

论证过程：丘奇论证了λ可定义函数类与一般递归函数类（哥德尔-赫尔布兰德递归函数）等同，且这个类恰好涵盖了所有"可有效计算"的函数。这意味着λ演算虽然形式极其简洁——仅有变量、λ抽象和函数应用三种构造——却足以表达任何可计算的函数。这一结论后来被图灵独立证明为等价于图灵机的计算能力，构成了"丘奇-图灵论题"的基础。

**甲译（白话意译）：** **所有"能算出来"的函数，都能用λ演算写出来。反过来说，λ演算能算的东西，跟递归函数能算的东西、跟图灵机能算的东西，完全一样。这就意味着，λ演算虽然只有三样零件——变量、造函数、用函数——但它什么都能算。它就是一个"万能计算机"，只不过是用纸上的符号来运转的。**

乙译（学术直译）：λ可定义正整数函数类与递归函数类相同，且此类包含一切其值可被有效计算之函数。λ演算乃一普遍之计算模型：凡可有效计算之函数皆可以λ表达式表之。

### 2.2 判定问题的不可解性（Undecidability of the Entscheidungsproblem）

> **"The general case of the Entscheidungsproblem of the pure first-order functional calculus is unsolvable."**
>
> **"There is no algorithm by means of which, given any formula of the pure first-order functional calculus, it can be determined whether or not the formula is provable."**
>
> **"We define a function to be effectively calculable if it is λ-definable. The Entscheidungsproblem asks whether there exists an effective procedure to determine provability in first-order logic. Since the set of λ-definable functions is not effectively enumerable in the required sense, no such procedure exists."**

论证过程：希尔伯特的判定问题（Entscheidungsproblem）问的是：是否存在一个机械算法，对于任意一阶逻辑公式，都能判定它是否可证（即是否为有效式）？丘奇的证明策略如下：

1. 首先定义"可有效计算的函数"为λ可定义的函数。
2. 构造一个特定的λ演算表达式，使得"判断该表达式是否有归约到某个范式"等价于一个已知的不可判定问题。
3. 证明如果一阶逻辑的判定问题可解，则可以借此解决上述不可判定问题，产生矛盾。
4. 因此，一阶逻辑的判定问题不可解。

这一结果意味着，逻辑推理本身存在根本性的限度——不存在万能的"推理机器"能自动判定一切逻辑命题的真伪。这与哥德尔不完备性定理遥相呼应：哥德尔证明了 sufficiently strong 的形式系统不可能既完备又一致，丘奇则进一步证明了即便不考虑完备性，仅判定可证性本身就是不可计算的。

**甲译（白话意译）：** **希尔伯特问了一个大问题：能不能造一台"逻辑裁判机"，你随便丢给它一条逻辑命题，它就能告诉你这条命题到底对不对、能不能证出来？丘奇的回答是：不能。他用自己发明的λ演算证明了，如果真有这种万能裁判机，那它就能解决一个已知不可解的问题——这显然矛盾。所以，这种机器不可能存在。换句话说，逻辑推理没有终点站，你不可能把所有逻辑问题都自动化地搞定。**

乙译（学术直译）：纯一阶函项演算之判定问题之一般情形不可解。不存在如下之算法：任给一阶函项演算之公式，借此算法即可判定该公式是否可证。吾人定义一函数为可有效计算者当且仅当其为λ可定义者。判定问题所问者，乃是否存在一有效程序以判定一阶逻辑中之可证性。鉴于λ可定义函数集在所需意义上非可有效枚举者，故此种程序不存在。

### 2.3 丘奇-图灵论题（Church-Turing Thesis）

> **"We now define the notion, already discussed, of an effectively calculable function of positive integers by identifying it with the notion of a λ-definable function."**
>
> **"This definition is thought to be justified by the considerations which follow, so far as positive justification can ever be achieved."**
>
> **"The same class of functions is obtained when the notion of effective calculability is identified with Turing computability."** (Church, citing Turing's independent result)

论证过程：丘奇-图灵论题并非一个可以证明的数学定理，而是一个关于"可计算性"概念的定义性主张。其论证逻辑如下：

1. 在1930年代之前，"可有效计算的函数"是一个直觉性概念，缺乏精确的数学定义。
2. 丘奇提出：将"可有效计算"定义为"λ可定义"。
3. 图灵独立提出：将"可有效计算"定义为"图灵机可计算"。
4. 丘奇随后证明：λ可定义函数类 = 图灵机可计算函数类 = 一般递归函数类。
5. 三种截然不同的形式化方案得出同一结果，这为论题提供了强有力的经验支持。

论题的哲学意义在于：它将一个模糊的直觉概念（"能算出来"）与精确的数学概念（λ可定义/图灵可计算）等同起来，从而为"什么是计算"提供了一个确切的标准。此后一切关于可计算性的讨论，都以这一论题为出发点。

**甲译（白话意译）：** **什么叫"能算出来"？这个问题听起来简单，其实很难回答——你怎么知道一个函数一定能算出来？丘奇说：咱就把"能算出来"定义成"能用λ演算写出来"。图灵说：我定义成"能用我的机器算出来"。结果发现，这两个人说的是同一回事——能写进λ演算的，恰好就是图灵机能算的，也恰好就是递归函数能表达的。三殊途同归，说明他们抓住了"计算"的本质。这不是定理，没法证明，但所有人都信它。**

乙译（学术直译）：吾人今将正整数可有效计算函数之概念——前已论及——定义为λ可定义函数之概念。此定义据以下诸考量以为正当，尽积极之正当性所能达者。当将有效可计算性之概念等同于图灵可计算性时，所得之函数类相同。

## 三、关键术语脚注

[^church1]: **λ演算（Lambda Calculus）**：丘奇于1930年代提出的一种形式系统，以函数的抽象（abstraction）与应用（application）为基本操作，仅含变量、λ抽象与函数应用三种构造，却可表达一切可计算函数。

[^church2]: **λ抽象（Lambda Abstraction）**：构造函数的基本操作，形如 λx.M，表示以 x 为参数、函数体为 M 的匿名函数。

[^church3]: **β归约（β-Reduction）**：λ演算中的基本计算步骤，即将 (λx.M) N 中 M 内自由出现的 x 替换为 N，记作 (λx.M) N →β M[x:=N]。

[^church4]: **丘奇数（Church Numerals）**：将自然数 n 编码为高阶函数 λf.λx.fⁿ(x) 的方案，使得算术运算可纯以λ演算表达，无需预设"数"的概念。

[^church5]: **判定问题（Entscheidungsproblem）**：希尔伯特提出的判定问题：是否存在一个机械算法，对于任意一阶逻辑公式，判定其是否为有效式（即可证）。丘奇于1936年证明该问题不可解。

[^church6]: **丘奇定理（Church's Theorem）**：一阶逻辑的判定问题不可解，即不存在算法能判定任一一阶逻辑公式是否可证。此结果确立了逻辑推理的根本限度。

[^church7]: **丘奇-图灵论题（Church-Turing Thesis）**：每一可有效计算的函数都是λ可定义的（等价地，图灵机可计算的）。此论题将直觉性的"可计算"概念精确化，是理论计算机科学的基石。

[^church8]: **丘奇-罗瑟定理（Church-Rosser Theorem）**：λ演算的汇合性定理：若一个λ表达式可归约到两个不同范式，则这两个范式必可进一步归约到同一范式。这保证了λ演算中计算结果的唯一性。

[^church9]: **一般递归函数（General Recursive Functions）**：哥德尔-赫尔布兰德定义的递归函数类，丘奇证明其与λ可定义函数类等同，二者共同构成"可有效计算函数"的数学刻画。

[^church10]: **可判定性（Decidability）**：一个问题称为可判定的，当且仅当存在一个算法（在有限步骤内停机并给出肯定或否定回答的程序）来解决它。丘奇定理表明一阶逻辑的有效性问题是不可判定的。

[^church11]: **范式（Normal Form）**：在λ演算中，一个不能再进行β归约的表达式称为范式。并非所有λ表达式都有范式，这与停机问题密切相关。
